# Test Report — OpenLibrary API

## 1. Введение
Данный отчёт отражает результаты тестирования публичного API OpenLibrary.  
Цель тестирования: оценить корректность и стабильность эндпоинтов `/search.json` и `/isbn/{isbn}.json`, выявить дефекты при стандартном и нестандартном вводе, проверить безопасность и производительность API.

## 2. Объём тестирования (Scope)
**Что было протестировано:**  
- Поиск по параметру `q`: валидные и невалидные строки, спецсимволы, Unicode, минимальная и максимальная длина, отсутствие параметра.  
- Эндпоинт `/isbn/{isbn}.json`: валидные, невалидные и слишком длинные ISBN.  
- Rate limiting: последовательные запросы.  
- Безопасность: SQL Injection, XSS, Path Traversal.  

**Что не тестировалось:**  
- Авторизация и аутентификация (API публичное).  
- Тестирование фронтенда OpenLibrary.  
- Интеграция с внешними сервисами, кроме поиска и ISBN.

## 3. Методы и подходы
- Ручное тестирование по подготовленным тест-кейсам (`testcases.md`).  
- Автотесты с использованием Postman и Python (requests).  
- Exploratory-тестирование для нестандартных сценариев, fuzzing.  
- Security-тесты на SQLi, XSS, Path Traversal.  

## 4. Результаты тестирования
- Количество тест-кейсов: 26  
- Пройдено (PASSED): 23  
- Провалено (FAILED): 2 (BUG-01, BUG-02)  
- Допустимо (ACCEPTABLE): 1 (POST вместо GET)  
- Заблокировано: 0  

## 5. Обнаруженные дефекты
| ID | Название | Severity | Статус |
|----|----------|---------|--------|
| BUG-01 | 500 Internal Server Error на спецсимволах в `q` | High | Open |
| BUG-02 | 422 Unprocessable Entity при `q < 3` | Low | Open |
| BUG-03 | XSS возвращается как строка | Medium | Open |

## 6. Качество продукта
- API стабилен при стандартных запросах и валидных данных.  
- Критические дефекты: 500 Internal Server Error на спецсимволах (влияет на стабильность).  
- Продукт можно использовать для нормальных сценариев, но рекомендуется исправление критических багов и документация ограничений параметров.  

## 7. Риски и проблемы
- Возможные падения при нестандартном вводе.  
- Недокументированные ограничения по длине `q`.  
- Возможные изменения публичного API OpenLibrary без уведомления.  

## 8. Выводы и рекомендации
- Исправить обработку спецсимволов и убрать утечку stacktrace.  
- Задокументировать минимальную длину и допустимые символы для параметра `q`.  
- Добавить fuzzing для комплексного тестирования ввода.  
- Проверить интеграцию исправлений через регрессионные тесты.
